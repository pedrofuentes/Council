### Fixed

- **`council sessions` now flags stale running debates with a recovery hint** (T2) — when a running debate has had no activity for more than one hour, `council sessions` now points users to `council resume <panel>` so they can recover the conversation instead of assuming it is still actively progressing.
- **Abort path no longer shows raw engine errors** (#810) — when a debate is interrupted via Ctrl+C, `runWithEngine` now suppresses confusing `formatEngineError` diagnostic output and returns normally, allowing callers to show the user-friendly "Debate interrupted. Partial results saved." message regardless of where in the lifecycle the abort occurred.
- **Config lock handles Windows NTFS contention** (#820) — `withConfigLock` now retries on `EPERM` and `EACCES` in addition to `EEXIST`, fixing intermittent crashes when concurrent processes update config simultaneously on Windows. Temp directories in config tests moved from `process.cwd()` to `os.tmpdir()` to prevent orphaned dirs in the workspace.

- **Expert creation now trusts `<slug>.yaml` over stale cache rows** (T2) — `council expert create --slug <slug> ...` now treats the YAML file as the source of truth for slug conflicts. If the YAML is missing but a stale `expert_library` row remains, Council recreates the YAML, refreshes the cached metadata row, and emits a warning that stale cache recovery occurred.
- **`council export` matches panel names by prefix** (T7) — `council export <prefix>` now resolves a unique-prefix match the same way `council resume` does. If the prefix is ambiguous, the matching names are listed and the command exits with a clear error. Exact-name lookup remains backward compatible.
- **`council export` includes the full history of resumed sessions** (T7) — previously only the most substantive single debate was exported, so resumed panels lost their original rounds. Export now flattens every debate (original + each resumption) into one continuous transcript with globally renumbered rounds; markdown / json / adr formats are unchanged.

- **Memory entries sanitised in `renderMemory()`** (#546, T-03) — `positions`, `updatedPriors`, and `unresolved` strings are now run through `sanitizePromptField` before being interpolated into the `[N] MEMORY` section of the system prompt. - **Context-manager `sanitizeFenceField` consolidated; speaker names sanitised** (#545, T-07) — `src/core/chat/context-manager.ts` previously carried its own private fourth copy of `sanitizeFenceField`; it now imports the shared `escapeFenceContent` from `prompt-sanitize.ts`. - **Recency weighting for persona document analysis** (Roadmap 6.8) — `analyzeDocuments()` (`src/core/documents/profile-analyzer.ts`) now annotates each document block in the analyzer meta-prompt with a recency-weight tag (`[Weight: 0.NN]`) computed from the document's `modifiedAt` timestamp via exponential decay: `weight = 2^(-ageDays / halfLifeDays)` (so age = 0 → 1.0, age = halfLife → 0.5, age = 2× halfLife → 0.25). - **On-demand persona document processing** (Roadmap 6.4) — when you `council chat <persona-slug>`, Council now auto-detects new, modified, or deleted files in `~/Council/experts/<slug>/docs/`, extracts and indexes new/changed files, reconciles deletions (the FTS index is pruned and the DB row is marked removed), refreshes the persona profile, and only then registers the expert with the engine — so subsequent turns immediately benefit from the latest reference material. Idempotent (unchanged docs are skipped via SHA-256 checksum), incremental (only new/changed files are re-extracted), and non-blocking (per-file failures are reported as warnings without aborting the chat; analyzer failures surface a one-line warning and the previous profile is preserved). Empty docs folders surface a one-line info message and the persona runs as a generic expert. Generic (non-persona) experts are unaffected. **Security:** the docs root must be a real directory — symlinks/junctions as the root itself are rejected up front, and per-file confinement uses a once-resolved canonical root passed through the detector and extractor (closing root-swap TOCTOU windows). Implementation: `createDocumentProcessor` in `src/core/documents/processor.ts`; integration in `runExpertChat` in `src/cli/commands/chat.ts`.
- **TOCTOU-hardened document extractor** — `extractDocument` now performs fd-based reads (open → fd-stat → realpath → lstat compare → confinement check → read via handle) so a symlink swap between path resolution and read cannot exfiltrate data outside the configured `confinementRoot`. Back-compat preserved when `confinementRoot` is omitted.

- **@mention routing in panel chat** (Roadmap 5.5) — type `@<expert-slug> <message>` to direct a question to one or more specific panelists; only the named experts respond, with each persisted chat turn carrying an `isMention: true` flag. Multiple mentions are supported (`@cto @sre …`) and experts respond in the panel's declared order regardless of mention order. Unknown slugs surface a one-line error listing the available panel slugs and the malformed user turn is NOT persisted, so the prompt can be retyped cleanly. Non-mentioned panelists still see the exchange in their context on subsequent turns. The parser is bypassed in 1:1 expert chat (a literal `@x …` is processed as plain text). Implementation: `parseUserInput()` in `src/core/chat/mention-parser.ts`.

- **`@convene <topic>` — inline structured debate** (Roadmap 5.6) — within an active panel chat, type `@convene <topic>` to interrupt the freeform conversation with a structured 4-phase deliberation (opening → cross-examination → rebuttal → synthesis; cross-exam skipped for single-expert panels) using the same engine and registered experts. Each debate `turn.end` is persisted as a regular `ChatTurn` so the deliberation joins the chat history and is visible to all subsequent turns. After the debate completes (or completes-with-partial-failures), control returns to freeform chat with a confirmation or interruption notice. Per-turn engine errors are non-terminal; partial results are preserved on interruption.

- **Template migration to the library format** (Roadmap 4.6) — on the first `council convene --template <name>` run after upgrade, Council automatically extracts the inline experts from each built-in panel (`architecture-review`, `career-coaching`, `code-review`, `incident-postmortem`, `startup-validation`) into standalone YAMLs under `~/Council/experts/<slug>.yaml` and rewrites each panel into `~/Council/panels/<name>.yaml` referencing those experts by slug. Migration is **non-destructive** (it never overwrites an existing user expert or panel YAML), **idempotent** (safe to re-run; short-circuits once both the filesystem and the `expert_library` AND `panel_library` SQLite tables show migrated state), and **DB-reset-resilient** (if the on-disk YAMLs survive a DB wipe of either the expert or panel tables, the next run re-registers library rows from disk, refreshes stale `panel_library` metadata from user-edited panel YAML, and materialises any inline expert overrides into `expert_library` so `panel_members` FK constraints are satisfied — all without rewriting files). Skips turn.end for unmapped expert slugs (graceful). Foundational for ROADMAP §3.1 persistent-memory and §3.2 session-resume.
- **`council convene <topic> --template <name> --engine <kind>`** — runs a panel debate end-to-end. Loads template → builds 8-section system prompts → opens DB → inserts panel + experts → constructs Debate over the chosen engine → wraps in DebatePersister → renders via JSON or Plain renderer. Supports `--mode freeform|structured`, `--max-rounds`, `--max-words`, `--format json|plain`. **Engine selection is explicit (no silent default)**: `--engine mock` for offline deterministic runs (prints a prominent `[MOCK ENGINE]` warning, tags the persisted debate with `engine='mock'`); `--engine copilot` runs against the real Copilot SDK.

### Changed

- ADR-005 supersedes the implicit `better-sqlite3` choice from ADR-002 / ROADMAP §1.7. The persistence backend is now `@libsql/client` (pure WASM) + `@libsql/kysely-libsql`. Rationale: `better-sqlite3` requires native build tools and lacks Node 25.5.0 prebuilds, breaking "simple to run". libsql is pure JS, has an official Kysely dialect, and is API-compatible with Turso (Council Cloud Phase 5).
- `council convene --engine copilot` now wires the real `CopilotEngine` (previously threw "not yet wired"). Integration tests gated by `COUNCIL_INTEGRATION=1` env var (off by default for fast unit-only runs).
- `makeEngineFromKind(kind)` exported from `src/cli/commands/convene.ts` so unit tests can verify wiring without invoking the engine.

### Hardening

This release lands a wave of defensive fixes from the post-Phase-7 Sentinel sweep. None changes user-facing behavior on the happy path; all close gaps that previously could leave Council in a half-consistent state, leak data across confinement boundaries, or surface generic errors instead of actionable guidance.

- **Atomic operations for non-atomic DB paths (#382, #383, #356, #363, #387, #388)** — six write paths that previously issued multiple statements without a surrounding transaction (panel deletion + cascade cleanup, expert deletion + panel-member detach, document indexer FTS sync, persona profile upsert + checksum update, panel docs unlink, chat session archive) are now wrapped in `BEGIN`/`COMMIT`/`ROLLBACK` on the libsql connection. A failure mid-sequence rolls back cleanly instead of leaving orphan rows or stale FTS entries; user-facing confirmation lines fire only after a successful `COMMIT`.
- **Profile analyzer sanitization (#358, #366)** — `existingProfile` fields injected into the analyzer meta-prompt now flow through the shared `sanitizePromptField` (extracted to `src/core/prompt-sanitize.ts`) before the fence-specific `<` → `&lt;` escape: C0 control bytes stripped, Unicode line breaks (NEL / U+2028 / U+2029) and CR/LF runs collapsed, bracketed `[N]` section markers defanged to `(sec-N)`, length capped at 2000 chars. Prevents a hostile persisted profile field from forging section headers in the analyzer prompt.
- **Panel document security — confinement, stale-file pruning, symlink consistency (#385, #386, #389, #390)** — linked-folder paths are canonicalised once and passed as `confinementRoot` to BOTH the detector and the extractor (defense-in-depth against detector→extractor TOCTOU symlink swaps); `panel docs link` uses `fs.lstat` and rejects symlinked paths up front so the link command and the scanner agree on confinement policy. Stale `panel_documents` rows whose files have disappeared from disk are pruned with their FTS entries on every scan, but transient folder-level failures (unmounted drive, permission flicker) explicitly do NOT trigger pruning so a temporary outage cannot wipe the index.
- **Memory recall for persona experts (#364)** — `recallMemory()` previously hardcoded the `[7]` section index when matching persisted system prompts, so when a persona expert had a `[N] PERSONA PROFILE` section that shifted `MEMORY` to `[8]`, the recall patch silently no-op'd. Section numbering is now derived from the actual prompt structure (the patch path locates the section header via `lastIndexOf` rather than a fixed offset) so persona experts get their recalled memory injected correctly.
- **Error handling improvements — timeouts, stream error rejection, transaction-wrapped reset (#359, #360, #361, #341, #342, #403)** — engine `send()` calls in `conclude` (manual `AbortController` + `setTimeout` in `src/cli/commands/conclude.ts`) and the profile analyzer (`AbortSignal.timeout` in `src/core/documents/profile-analyzer.ts`) now bound the wait so a hung backend fails fast with an actionable message instead of hanging indefinitely; stream `error` events are surfaced as rejected promises rather than silently swallowed; and `council memory reset` (default mode) is wrapped in a single `BEGIN`/`COMMIT`/`ROLLBACK` so a failure between the debate-delete and expert-memory-clear loops cannot leave the panel half-reset.

### Fixed

- **`council memory reset` (default mode) is now atomic (#403)** — the default reset path previously deleted every debate and then cleared each expert's `extractedMemoryJson` as independent statements with no surrounding transaction, so a failure between or during either loop left the panel half-reset (debates gone but expert memory preserved, or vice versa) with no recovery. Both loops are now wrapped in a single `BEGIN`/`COMMIT`/`ROLLBACK` on the libsql connection (matching the convention used by `document-repository.clearForRetrain` and the documents indexer — Kysely's `db.transaction()` reconnects libsql and drops FTS5 virtual tables on `:memory:`). The user-facing confirmation lines are now emitted only after a successful `COMMIT`, so a rolled-back reset is never partially announced. If `ROLLBACK` itself fails, the original error is still rethrown and a `!! ROLLBACK failed …` line is written to stderr so operators know the DB may need a manual check. Implementation: `buildResetCommand` in `src/cli/commands/memory.ts`.
- **`council expert train --retrain` honest rollback messaging (#383, #425)** — `DocumentRepository.clearForRetrain` now throws a typed `ClearForRetrainError` whose `rollbackFailed: boolean` field records whether `ROLLBACK` itself succeeded, alongside the original `cause` and any `rollbackError`. The `BEGIN` statement is also wrapped so a connection-level failure before any mutation still surfaces through the same typed contract (with `rollbackFailed: false`). The CLI now defaults conservatively: it only emits "Existing profile and tracking preserved" when it receives a `ClearForRetrainError` with `rollbackFailed === false`; any other error type — or a rollback that itself failed — produces a "database may be in an inconsistent state … inspect the document_index and expert_documents tables before retrying" message instead. Previously the CLI swallowed `ROLLBACK` failures and unconditionally claimed preservation, which could falsely reassure users that the DB was intact when in fact the FTS index and tracked documents may have disagreed.
- **Persona profile analyzer: existingProfile fields injected unsanitized (#358, #366)**— `analyzeDocuments()` now runs all `existingProfile` fields (`communicationStyle`, `decisionPatterns`, `biases`, `vocabulary`, `epistemicStance`) through the shared `sanitizePromptField` (extracted to `src/core/prompt-sanitize.ts` and reused by `src/core/prompt-builder.ts`) before interpolating them into the analyzer meta-prompt. C0 control bytes are stripped, Unicode line breaks (NEL / U+2028 / U+2029) and CR/LF runs are collapsed to a single space, bracketed numeric section markers (`[N]`) are defanged to `(sec-N)`, and each field is length-capped at 2000 characters. The fence-specific `<` → `&lt;` escape is layered on top so a persisted field still cannot prematurely close the `<documents>` data fence. Adversarial regression tests cover fence-break document content, `[N]`-marker spoofing, runaway-length fields, and C0/Unicode-line-break payloads.
- **`DebatePersister` single-use enforcement (#120)** — calling `persist()` a second time on the same instance now throws with a clear "single-use" message instead of silently overwriting `#debateId` and leaking `#pendingTurnPosition` state across debates. Construct a fresh persister per debate.
- **`DebatePersister` orphan turn.end warning (#119)** — `turn.end` events with no matching prior `turn.start` (orchestrator protocol violations) now call the optional `deps.logger.warn(...)` if a logger is configured. When no logger is provided, drops silently as before (backwards-compatible default).
- New `DebatePersisterLogger` interface (`{ warn(message: string): void }`) — minimal injectable surface for warnings; production callers can pass a console-backed implementation, tests pass a Vitest mock.
- **`DebatePersister.persist()`** no longer silently mutates the debate row to `status='aborted'` when the terminal `debate.end` update itself throws. Renamed `normalEnd` to `terminalUpdateAttempted` and set it BEFORE the await — so a failed terminal update lets the original error bubble (truthful) and leaves the row at its pre-attempt state (`running`). Resolves #150.
- **`convene` partial-failure rollback** now filters to only the experts whose `addExpert` actually fulfilled, instead of sweeping the whole experts list and relying on unverified `removeExpert`-idempotency. Adds an explicit JSDoc contract on `CouncilEngine.removeExpert` codifying the no-op-for-unknown-ids guarantee + a regression test against `MockEngine`. Resolves #151.
- `MockEngine.failOnAddExpert.afterN` semantics changed: now fails ALL calls after the Nth (was: only the (N+1)th). Sharper test signal — `afterN: 1` with 4 experts now means 1 fulfilled + 3 rejected, not 1+1+2.
- `MockEngine.removeExpertCalls: readonly string[]` — new test-only accessor capturing every `removeExpert(id)` invocation in temporal order.
- `convene` MOCK warning now writes to a separate `writeError` channel (stderr), keeping `--format json` output as pure NDJSON on stdout. Resolves #127.
- `makeEngineFromKind` now has an exhaustive `default` arm that throws on unknown engine kind (previously silently returned `undefined`). Resolves #128 and #134.
- `convene` `--engine` validation now uses validate-then-assign (no silent ternary fallback). Resolves #129.
- `convene` cleanup errors (engine.stop / db.destroy) are now surfaced via `writeError` with a `!! ... failed during cleanup:` prefix instead of being silently swallowed. Resolves #130.
- `convene` registers experts in parallel via `Promise.all` instead of sequentially, cutting startup latency from O(N × session-create-ms) to O(1). Resolves #131.
- `bin/council.ts` JSDoc updated — engine is no longer "mock by default". Resolves #126.
- `makeEngineFromKind` JSDoc tagged `@internal` per stability convention. Resolves #135.
- Removed redundant `if (!INTEGRATION) describe(...)` fallback in `tests/integration/convene-copilot.test.ts`. Resolves #136.
- **CopilotEngine + MockEngine** now expose a `lastStopErrors: readonly Error[]` getter populated during `stop()`. Per-session disconnect failures (and the final `client.stop()` failure) are aggregated here instead of being silently swallowed. Backwards-compatible: `stop()` still returns `Promise<void>`. Resolves #143.
- **`convene` parallel addExpert** is now leak-safe via `Promise.allSettled` + rollback. If any `addExpert` rejects, every successfully-created expert is removed via `engine.removeExpert()` before re-throwing, so no `CopilotSession` survives past cleanup. Error message: `could not register all experts (X/Y failed): <first reason>`. Resolves #142.
- **`DebatePersister.persist()`** now self-finalizes on abnormal exit. If the source stream throws or the consumer breaks the for-await loop, the debate row is marked `status='aborted'` with `endedAt` set in a `finally` block. This unblocks ROADMAP §3.2 session-resume which needs to distinguish abandoned from running debates. Resolves #117.
