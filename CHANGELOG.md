## [Unreleased]

### Added

- **Panel composition model** (Roadmap 4.2) ‚Äî `PanelDefinitionSchema` now accepts either a slug string (referencing the expert library from P4-T2) or an inline `ExpertDefinition` for each `experts` entry, so a single panel YAML can mix reusable library experts with one-off inline definitions. New loader functions:
  - **`loadUserPanel(name, dataHome)`** ‚Äî reads a user-authored YAML from `<dataHome>/panels/<name>.{yaml,yml}` (path-traversal-guarded; throws a typed `PanelNotFoundError` when absent so callers can fall back without parsing error strings).
  - **`listUserPanels(dataHome)`** ‚Äî returns user panel names without extension, sorted; empty array on a fresh install.
  - **`loadPanel(name, dataHome)`** ‚Äî user panels in `<dataHome>/panels/` take precedence over the built-in templates that ship with Council, so a user can override a stock panel by name without touching the package.
  - **`resolveExperts(entries, library)`** ‚Äî looks slug refs up in an `ExpertLibrary`, passes inline definitions through unchanged, and reports any missing slugs in a single collection so callers surface one useful error.
  - **`council convene --template <name>`** now resolves slug references against the expert library and reports unresolved slugs (with `'council experts create'` guidance) in a single clear error. Inline-only panels keep the existing fast path. Auto-composed panels reject slug strings from the composer (a non-inline response now aborts auto-compose with a descriptive error rather than silently producing an empty roster).
- **LLM-based ExpertMemory extraction** (ROADMAP ¬ß3.1) ‚Äî after each debate, an LLM extraction pass distils each AI expert's prior turns (bounded by the most-recent 5 debates √ó 20 turns) into structured `{ positions, updatedPriors, unresolved }` and caches the result on the expert row (new `experts.extracted_memory_json` column, migration 003). `recallMemory()` now prefers the cached LLM-distilled memory over the heuristic sentence scan, falling back to the heuristic when the cache is null or all-empty. Cost ceiling: one LLM call per AI expert per debate. Prompt-injection hardened: every `<` in interpolated turn content is escaped to `&lt;` before fencing (same pattern as the LLM summariser).
- **`council convene --heuristic-memory`** and **`council resume --continue --heuristic-memory`** ‚Äî opt-out flag that skips the post-debate LLM extraction pass and keeps the legacy heuristic-only recall behaviour. Useful for offline tests and air-gapped environments.

- **Context window management** (ROADMAP ¬ß2.6, closes #229-ish; see PR #249) ‚Äî keeps long debates within model limits without sacrificing fidelity. Three knobs, all opt-in:
  - **`council convene --context-scope <scope>`** ‚Äî `all` (default, unchanged), `same-round` (each expert only sees turns from the current round, including intra-round turns that completed just before), or `recent` (a sliding window of the most-recent N turns across all rounds, default N=10). Backed by `filterPriorTurns()` in `src/core/context/visibility.ts`.
  - **`council convene --summarize-after <n>`** ‚Äî once round `n` starts, the orchestrator prepends a compact heuristic rolling summary (each expert's first-sentence position + a disagreement-signal flag) to every prompt so the strategy can reference earlier deliberation without every verbatim turn. Backed by `buildRollingSummary()` in `src/core/context/summarizer.ts`.
  - **`DebateConfig.contextConfig.maxPromptChars`** (engine-level; no CLI flag yet) ‚Äî hard cap on verbatim prior-turn content. When the cap would be exceeded, oldest turns are dropped first; the most recent turn is always retained. Truncation is **opt-in** ‚Äî omitting `contextConfig` (or omitting `maxPromptChars`) preserves the legacy passthrough behaviour with no implicit cap.
  - Under any `contextConfig` the orchestrator re-plans the strategy _per turn_ so `same-round` and `recent` policies see turns that completed earlier in the same round. Without `contextConfig` the strategy is still planned once per round (legacy behaviour).
- **Rich Ink Terminal UI** (ROADMAP ¬ß3.4) ‚Äî React + Ink-based renderer that auto-activates when stdout is a TTY. Each expert is assigned a stable color from a 6-color palette (cyan, yellow, magenta, green, blue, red) so the same expert renders in the same color across every round. Components include a `PanelRoster` (assembled experts), `RoundHeader` (round separator), `ExpertCard` (colored name header), `StreamingText` (live turn body with caret cursor while writing), `RetryIndicator` (`ink-spinner` + reason while a turn is being retried), `CostIndicator` (premium-request budget), and a completion banner on `debate.end`. The selector lives in `src/cli/renderers/select.ts` and exposes a new `--format auto` value (the new default). Explicit `--format json` and `--format plain` still always override TTY detection so piped output stays deterministic. Adds `ink`, `ink-spinner`, and `react` as runtime dependencies.
- **`council convene --strategy <name>`** and **`council resume --continue --strategy <name>`** (ROADMAP ¬ß2.3, closes #212) ‚Äî wires `ModeratorStrategy` into the freeform debate orchestrator. Supported values: `round-robin` (default), `devils-advocate`, `consensus-check`. The `devils-advocate` strategy accepts an optional `:<expert-slug>` suffix to pin the contrarian (e.g. `--strategy devils-advocate:senior`); when omitted, the first expert is the advocate. The strategy controls per-round turn order _and_ per-turn prompts: round 0 gets the strategy's framing, round 1+ receives a `Prior discussion:` block built from earlier turn content. `consensus-check` can also terminate a debate early via `shouldContinue()` ‚Üí `debate.end { reason: "consensus" }`. Strategies are ignored in `--mode structured` (phase-prompts continue to drive that mode). The chosen strategy name is persisted to `debates.moderator`.
- **`DebateConfig.strategy?: ModeratorStrategy`** ‚Äî pluggable hook on the `Debate` orchestrator's freeform path. Defaults to `createRoundRobinStrategy()` so existing callers see the same structural event sequence.
- **`council conclude [panel] --engine <kind> [--format json|plain]`** (ROADMAP ¬ß2.7) ‚Äî synthesizes the latest debate of a panel into a structured decision framework (consensus, tensions, decision matrix, recommendation, confidence). Reads transcript from the local DB, runs a single moderator-style synthesis prompt through a transient "synthesizer" expert, parses the JSON response (tolerates `json` fences and prose-wrapped JSON), validates against a Zod schema, and renders plain or JSON. Defaults to the most recently-created panel when no name is given. Transcript content is delimited in `<transcript>` tags and capped at 50 turns / 50,000 chars to mitigate prompt injection and unbounded context. Synthesis call uses a 60s `AbortSignal.timeout` so a hung engine fails fast. Warnings are emitted when the latest debate isn't `completed` or when the transcript is truncated.
- **Expert memory recall** (ROADMAP ¬ß3.1, second half) ‚Äî `recallMemory()` reads an expert's prior turns from the local SQLite store and synthesises an `ExpertMemory` (positions, updatedPriors, unresolved) that the prompt-builder renders into Section [7] of the system prompt. `convene` recalls from the most recent prior panel sharing the same template; `resume --continue` patches the persisted system prompt via `applyRecalledMemory()`. Recall is per-expert-slug and parallelised across the roster (`Promise.all`).
- **Prompt-injection hardening for recalled memory** ‚Äî `sanitizeMemorySnippet()` strips section-header markers (`[N] ...`) and flattens line breaks from every recalled snippet before it reaches the prompt. `applyRecalledMemory()` uses `lastIndexOf("\n[8] CURRENT TASK")` so injected copies of the marker cannot extend the section boundary.
- **Panel auto-composition** (ROADMAP ¬ß2.5) ‚Äî `autoComposePanel(topic, engine, options?)` in `src/core/auto-compose.ts` registers a temporary "composer" expert primed with a meta-prompt, streams its JSON response, strips optional Markdown code fences, parses, and validates against `PanelDefinitionSchema`. Cleans up the composer expert via `finally` even on failure. Configurable `minExperts`/`maxExperts`/`defaultModel`. Surfaces engine error events, empty responses, malformed JSON, and Zod validation failures as descriptive errors. Foundation for `council convene "topic"` without `--template`.
- **`council convene --human "<name>"`** (ROADMAP ¬ß3.3) ‚Äî adds a human participant to the panel. Repeatable (`--human Alice --human Bob`). Human participants take turns alongside AI experts but input is collected interactively instead of via the engine. Human turns are persisted with `speakerKind: "human"` and do NOT count as premium requests in cost tracking. PlainRenderer shows `[You]` prefix for human turns.
- **`speakerKind` on DebateEvents** ‚Äî `turn.start`, `turn.delta`, `turn.end` now carry optional `speakerKind: "expert" | "human"` (defaults to `"expert"` for backwards compatibility). `DebatePersister` reads this from events instead of hardcoding.
- **`HumanInputProvider` interface** (`src/core/human-input.ts`) ‚Äî abstraction for collecting human input during debates. Returns `{ kind: "submitted", content }` or `{ kind: "cancelled", reason? }`. CLI implements via readline; tests inject canned responses.
- **`council ask <panel> "<question>" [--expert <slug>] [--engine <kind>]`** ‚Äî one-shot single-expert chat. Picks one expert from an existing panel (by slug or default to first), runs a 1-round 1-expert debate through the full pipeline (engine ‚Üí Debate ‚Üí DebatePersister ‚Üí Renderer), and returns. Creates a new debate row per ask visible in `resume`, `export`, `memory`. Uses `formatEngineError` for actionable CLI messages. Closes the last vapor-feature in the README.
- **Retry on recoverable engine errors** (ROADMAP ¬ß3.7) ‚Äî `Debate.#runTurn()` now automatically retries up to 2√ó with exponential backoff (`250ms`, `1s`) when `engine.send()` yields an `error` event with `recoverable: true` (RATE_LIMITED, NETWORK). Non-recoverable errors fail fast. Retry exhaustion emits a final error event but does NOT terminate the debate ‚Äî the orchestrator moves on to the next expert.
- **`turn.retry` `DebateEvent` variant** ‚Äî `{ kind: "turn.retry"; expertSlug; attempt; reason }`. Renderers MAY display `[expert retrying‚Ä¶]`; if ignored, downstream behavior is identical to a slightly slower turn. Persistence (`DebatePersister`) does NOT write a row for retries ‚Äî only the eventual `turn.end` (success) or `error` (exhaustion).
- **`DebateConfig.retryBackoffMs?: readonly number[]`** ‚Äî list of backoff delays in ms. Length determines max retries. Default `[250, 1000]`. Tests pass `[1, 2]` for fast suite runs.
- **`src/cli/error-mapper.ts`** ‚Äî `formatEngineError(err)` shared helper that maps `EngineErrorCode` to actionable user messages: NOT_AUTHENTICATED ‚Üí "run `gh auth login`", MODEL_UNAVAILABLE ‚Üí "model X isn't available; check `council doctor`", NETWORK ‚Üí "check your connection", RATE_LIMITED ‚Üí "wait and retry", CONTEXT_OVERFLOW ‚Üí "smaller `--max-words` or fewer experts", INTERNAL ‚Üí "file an issue at github.com/pedrofuentes/Council". Closes #133.
- **Wired into `convene` and `resume`** ‚Äî engine init + render path now wrapped in try/catch that routes errors through `formatEngineError` to stderr before re-throwing. Users see actionable hints instead of raw stack traces.
- New `MockEngine.failOnSend?: { expertId, afterN, failures, code, message }` test seam ‚Äî simulates per-send transient failures for retry testing.
- **`council memory list/inspect/reset`** (ROADMAP ¬ß3.5) ‚Äî three subcommands for inspecting and curating Council's local SQLite state at the panel/expert/debate/turn level.
  - **`memory list`**: per-panel summary (name, expert/debate/turn counts, last activity timestamp). `--panel <name>` filters to one. `--format json|plain`.
  - **`memory inspect <panel>`**: detailed view (topic, latest debate prompt + status + turn count, expert displayNames + models). `--expert <slug>` focuses on one expert and shows their (truncated, 600 chars) system prompt, per-expert turn count, and recalled memory (positions / updated priors / unresolved questions extracted from prior turns).
  - **`memory reset <panel> --yes`**: destructive. **Requires `--yes` flag explicitly** (no interactive prompt ‚Äî flag-only safety gate). Default mode deletes debates + turns, keeps panel + experts. `--hard` deletes the panel entirely (FK CASCADE removes everything). `--expert <slug>` drops one expert from the panel.
  - Pure DB read + targeted writes: no engine, no LLM. Wired in as the 7th subcommand.
- **`council export <panel> --format markdown|json|adr [--output <path>]`** (ROADMAP ¬ß3.6) ‚Äî snapshots the latest debate of a panel into a shareable artifact.
  - **markdown** (default): readable transcript with H1 header (panel name + topic), status line, panel roster, per-turn sections grouped by round (expert displayName + model + content as block quotes).
  - **json**: NDJSON identical to `council resume --format json` ‚Äî same `synthesizeEvents()` helper.
  - **adr**: Architecture Decision Record markdown with Status / Context / Options Considered / Discussion / Decision sections populated from the panel's debate (heuristic: opening turn per expert = position; last turn per expert = synthesis).
  - **`--output <path>`**: writes to file instead of stdout (default: stdout). Confirmation message goes to stderr so stdout stays clean.
- **`src/memory/transcript.ts`** ‚Äî `loadTranscript(db, panelName)` + `synthesizeEvents(doc)` extracted from `resume`. Pure read path: no engine, no LLM, no persistence side effects. Shared by `resume` and `export` so both commands produce identical event streams from the same DB rows.
- `TranscriptDocument` type ‚Äî `{ panel, experts, latestDebate, turns }`.
- **`council resume <panel>`** (ROADMAP ¬ß3.2) ‚Äî reopens an existing panel.
  - **Transcript mode** (default): replays the most recent debate's persisted turns as a synthesized `DebateEvent` stream and renders via JSON or Plain. No engine, no LLM calls ‚Äî pure DB read.
  - **Continue mode** (`--continue "<prompt>"`): runs a NEW debate against the same panel/experts using the existing convene wiring (engine + Debate + DebatePersister + Renderer). Honors the panel's persisted `mode` (freeform or structured) from `configJson`. Reuses the panel's stored expert system prompts and patches Section [7] MEMORY with freshly-recalled memory (¬ß3.1 second half).
  - Engine selection mirrors convene: `--engine mock|copilot` is **required** when `--continue` is set (no silent default ‚Äî production users never get fake debates persisted as real). Loud `[MOCK ENGINE]` warning to stderr fires only when `mock` is explicitly chosen.
  - Other options: `--format json|plain`, `--max-rounds`, `--max-words`.
  - Resolves panel name ‚Üí most-recently-created match via `PanelRepository.findByName()`.
- `PanelRepository.findByName(name)` ‚Äî look up the most-recently-created panel by name. Used by `council resume`. Returns `undefined` if no match.

### Changed

- ADR-005 supersedes the implicit `better-sqlite3` choice from ADR-002 / ROADMAP ¬ß1.7. The persistence backend is now `@libsql/client` (pure WASM) + `@libsql/kysely-libsql`. Rationale: `better-sqlite3` requires native build tools and lacks Node 25.5.0 prebuilds, breaking "simple to run". libsql is pure JS, has an official Kysely dialect, and is API-compatible with Turso (Council Cloud Phase 5).
- `council convene --engine copilot` now wires the real `CopilotEngine` (previously threw "not yet wired"). Integration tests gated by `COUNCIL_INTEGRATION=1` env var (off by default for fast unit-only runs).
- `makeEngineFromKind(kind)` exported from `src/cli/commands/convene.ts` so unit tests can verify wiring without invoking the engine.

### Fixed

- **`DebatePersister` single-use enforcement (#120)** ‚Äî calling `persist()` a second time on the same instance now throws with a clear "single-use" message instead of silently overwriting `#debateId` and leaking `#pendingTurnPosition` state across debates. Construct a fresh persister per debate.
- **`DebatePersister` orphan turn.end warning (#119)** ‚Äî `turn.end` events with no matching prior `turn.start` (orchestrator protocol violations) now call the optional `deps.logger.warn(...)` if a logger is configured. When no logger is provided, drops silently as before (backwards-compatible default).
- New `DebatePersisterLogger` interface (`{ warn(message: string): void }`) ‚Äî minimal injectable surface for warnings; production callers can pass a console-backed implementation, tests pass a Vitest mock.
- **`DebatePersister.persist()`** no longer silently mutates the debate row to `status='aborted'` when the terminal `debate.end` update itself throws. Renamed `normalEnd` to `terminalUpdateAttempted` and set it BEFORE the await ‚Äî so a failed terminal update lets the original error bubble (truthful) and leaves the row at its pre-attempt state (`running`). Resolves #150.
- **`convene` partial-failure rollback** now filters to only the experts whose `addExpert` actually fulfilled, instead of sweeping the whole experts list and relying on unverified `removeExpert`-idempotency. Adds an explicit JSDoc contract on `CouncilEngine.removeExpert` codifying the no-op-for-unknown-ids guarantee + a regression test against `MockEngine`. Resolves #151.
- `MockEngine.failOnAddExpert.afterN` semantics changed: now fails ALL calls after the Nth (was: only the (N+1)th). Sharper test signal ‚Äî `afterN: 1` with 4 experts now means 1 fulfilled + 3 rejected, not 1+1+2.
