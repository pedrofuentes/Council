## [Unreleased]

### Added

- **Cross-panel awareness in 1:1 chat** (Roadmap 7.2 + 7.3) — when you `council chat <persona-slug>`, the expert's system prompt now includes a `[PANEL MEMBERSHIPS]` section listing the other panels they belong to (with co-members and panel descriptions), so 1:1 conversations are informed by the shared context across panels. Memberships are ordered most-recently-active first (by `panel_library.updated_at`) and capped at 5 entries to bound prompt size. All externally-sourced fields (panel names, descriptions, co-member display names) are run through the same sanitiser used for persona profile fields — C0 controls stripped, line breaks collapsed, `[N]` section markers defanged to `(sec-N)`, length capped at 2000 chars — so a panel YAML cannot forge section headers in the privileged system prompt. The query is best-effort: a DB failure surfaces a one-line warning (not silent) and chat continues without the section. Panel chat is unchanged. Implementation: `getExpertPanelMemberships()` in `src/core/panel-membership-query.ts`, `renderPanelMemberships()` and `PanelMembership` in `src/core/prompt-builder.ts`.

- **Panel document folder** (Roadmap 6.7) — panels now have a shared reference-document corpus indexed into FTS5 alongside expert documents. Three new CLI subcommands:
  - `council panel docs <name>` — list managed and externally-linked folders for a panel and their document counts.
  - `council panel docs link <name> --path <folder>` — link an external folder so its documents are scanned and indexed at chat start.
  - `council panel docs unlink <name> --path <folder>` — remove a linked folder and clean up its `panel_documents` rows + FTS index entries. If FTS cleanup fails the unlink aborts with a non-zero exit and the linked-folder + `panel_documents` rows are preserved so the operation can be retried.

  `council panel create` auto-provisions `~/Council/panels/<name>/docs/` so users can drop shared docs without explicit linking. On `council chat <panel>`, the panel scanner walks both the managed and linked folders, detects new/changed files via SHA-256 checksums, extracts and indexes them under `source_type='panel'`, prunes any tracked documents that have disappeared from disk (both `panel_documents` rows are marked `removed` and the FTS `document_index` entries are deleted; transient folder-level failures do NOT trigger pruning, to avoid wiping the index on an unmounted drive), and surfaces a one-line summary plus a warning if any folder failed to scan (distinguishing the managed folder from linked folders). If every discovered document fails to process, a `⚠ All N documents failed to process. Check file formats and permissions.` warning is surfaced so the user is not left with a silent empty corpus. **Security:** linked folders are canonicalised once and passed as `confinementRoot` to BOTH the detector and the extractor (defense-in-depth against detector→extractor TOCTOU symlink swaps); `panel docs link` uses `fs.lstat` and rejects symlinked paths up-front so the link command and the scanner agree on the confinement policy. New schema in migration 009: `panel_linked_folders` and `panel_documents` (FK CASCADE from `panel_library`).

- **On-demand persona document processing** (Roadmap 6.4) — when you `council chat <persona-slug>`, Council now auto-detects new, modified, or deleted files in `~/Council/experts/<slug>/docs/`, extracts and indexes new/changed files, reconciles deletions (the FTS index is pruned and the DB row is marked removed), refreshes the persona profile, and only then registers the expert with the engine — so subsequent turns immediately benefit from the latest reference material. Idempotent (unchanged docs are skipped via SHA-256 checksum), incremental (only new/changed files are re-extracted), and non-blocking (per-file failures are reported as warnings without aborting the chat; analyzer failures surface a one-line warning and the previous profile is preserved). Empty docs folders surface a one-line info message and the persona runs as a generic expert. Generic (non-persona) experts are unaffected. **Security:** the docs root must be a real directory — symlinks/junctions as the root itself are rejected up front, and per-file confinement uses a once-resolved canonical root passed through the detector and extractor (closing root-swap TOCTOU windows). Implementation: `createDocumentProcessor` in `src/core/documents/processor.ts`; integration in `runExpertChat` in `src/cli/commands/chat.ts`.
- **TOCTOU-hardened document extractor** — `extractDocument` now performs fd-based reads (open → fd-stat → realpath → lstat compare → confinement check → read via handle) so a symlink swap between path resolution and read cannot exfiltrate data outside the configured `confinementRoot`. Back-compat preserved when `confinementRoot` is omitted.

- **@mention routing in panel chat** (Roadmap 5.5) — type `@<expert-slug> <message>` to direct a question to one or more specific panelists; only the named experts respond, with each persisted chat turn carrying an `isMention: true` flag. Multiple mentions are supported (`@cto @sre …`) and experts respond in the panel's declared order regardless of mention order. Unknown slugs surface a one-line error listing the available panel slugs and the malformed user turn is NOT persisted, so the prompt can be retyped cleanly. Non-mentioned panelists still see the exchange in their context on subsequent turns. The parser is bypassed in 1:1 expert chat (a literal `@x …` is processed as plain text). Implementation: `parseUserInput()` in `src/core/chat/mention-parser.ts`.

- **`@convene <topic>` — inline structured debate** (Roadmap 5.6) — within an active panel chat, type `@convene <topic>` to interrupt the freeform conversation with a structured 4-phase deliberation (opening → cross-examination → rebuttal → synthesis; cross-exam skipped for single-expert panels) using the same engine and registered experts. Each debate `turn.end` is persisted as a regular `ChatTurn` so the deliberation joins the chat history and is visible to all subsequent turns. After the debate completes (or completes-with-partial-failures), control returns to freeform chat with a confirmation or interruption notice. Per-turn engine errors are non-terminal; partial results are preserved on interruption.

- **Template migration to the library format** (Roadmap 4.6) — on the first `council convene --template <name>` run after upgrade, Council automatically extracts the inline experts from each built-in panel (`architecture-review`, `career-coaching`, `code-review`, `incident-postmortem`, `startup-validation`) into standalone YAMLs under `~/Council/experts/<slug>.yaml` and rewrites each panel into `~/Council/panels/<name>.yaml` referencing those experts by slug. Migration is **non-destructive** (it never overwrites an existing user expert or panel YAML), **idempotent** (safe to re-run; short-circuits once both the filesystem and the `expert_library` AND `panel_library` SQLite tables show migrated state), and **DB-reset-resilient** (if the on-disk YAMLs survive a DB wipe of either the expert or panel tables, the next run re-registers library rows from disk, refreshes stale `panel_library` metadata from user-edited panel YAML, and materialises any inline expert overrides into `expert_library` so `panel_members` FK constraints are satisfied — all without rewriting files). Slug collisions across panels are resolved by reusing identical experts and suffixing genuinely-different ones with the panel name (e.g. `sre-incident-postmortem`). Programmatic entry point: `migrateBuiltInTemplates(dataHome, library, db, options?)` and `isMigrationNeeded(dataHome, db?)` in `src/core/template-migration.ts`.

- **Panel composition model** (Roadmap 4.2) — `PanelDefinitionSchema` now accepts either a slug string (referencing the expert library from P4-T2) or an inline `ExpertDefinition` for each `experts` entry, so a single panel YAML can mix reusable library experts with one-off inline definitions. New loader functions:
  - **`loadUserPanel(name, dataHome)`** — reads a user-authored YAML from `<dataHome>/panels/<name>.{yaml,yml}` (path-traversal-guarded; throws a typed `PanelNotFoundError` when absent so callers can fall back without parsing error strings).
  - **`listUserPanels(dataHome)`** — returns user panel names without extension, sorted; empty array on a fresh install.
  - **`loadPanel(name, dataHome)`** — user panels in `<dataHome>/panels/` take precedence over the built-in templates that ship with Council, so a user can override a stock panel by name without touching the package.
  - **`resolveExperts(entries, library)`** — looks slug refs up in an `ExpertLibrary`, passes inline definitions through unchanged, and reports any missing slugs in a single collection so callers surface one useful error.
  - **`council convene --template <name>`** now resolves slug references against the expert library and reports unresolved slugs (with `'council experts create'` guidance) in a single clear error. Inline-only panels keep the existing fast path. Auto-composed panels reject slug strings from the composer (a non-inline response now aborts auto-compose with a descriptive error rather than silently producing an empty roster).
- **Expert library management CLI** (ROADMAP §4.3) — `council expert create | list | inspect | edit | delete` lets users author, browse, and curate the expert library at `~/Council/experts/*.yaml` (with parallel metadata in the `expert_library` SQLite table). `create` is an interactive readline wizard with a complete set of flags (`--slug`, `--name`, `--role`, `--expertise`, `--stance`, `--personality`, `--persona`, `--persona-description`) for non-TTY use; the `--persona` flag creates a `~/Council/experts/<slug>/docs/` folder for relationship documents. `edit` opens `$VISUAL`/`$EDITOR` (falls back to `vi`/`notepad`), then re-parses the file, refuses slug renames, and syncs the DB metadata row + checksum so library reads stay consistent with disk. `delete` refuses when the expert is a member of any panel unless `--force` is passed. `list` supports `--format table` (default) and `--format json`.


- **LLM-based ExpertMemory extraction** (ROADMAP §3.1) — after each debate, an LLM extraction pass distils each AI expert's prior turns (bounded by the most-recent 5 debates × 20 turns) into structured `{ positions, updatedPriors, unresolved }` and caches the result on the expert row (new `experts.extracted_memory_json` column, migration 003). `recallMemory()` now prefers the cached LLM-distilled memory over the heuristic sentence scan, falling back to the heuristic when the cache is null or all-empty. Cost ceiling: one LLM call per AI expert per debate. Prompt-injection hardened: every `<` in interpolated turn content is escaped to `&lt;` before fencing (same pattern as the LLM summariser).
- **`council convene --heuristic-memory`** and **`council resume --continue --heuristic-memory`** — opt-out flag that skips the post-debate LLM extraction pass and keeps the legacy heuristic-only recall behaviour. Useful for offline tests and air-gapped environments.

- **Context window management** (ROADMAP §2.6, closes #229-ish; see PR #249) — keeps long debates within model limits without sacrificing fidelity. Three knobs, all opt-in:
  - **`council convene --context-scope <scope>`** — `all` (default, unchanged), `same-round` (each expert only sees turns from the current round, including intra-round turns that completed just before), or `recent` (a sliding window of the most-recent N turns across all rounds, default N=10). Backed by `filterPriorTurns()` in `src/core/context/visibility.ts`.
  - **`council convene --summarize-after <n>`** — once round `n` starts, the orchestrator prepends a compact heuristic rolling summary (each expert's first-sentence position + a disagreement-signal flag) to every prompt so the strategy can reference earlier deliberation without every verbatim turn. Backed by `buildRollingSummary()` in `src/core/context/summarizer.ts`.
  - **`DebateConfig.contextConfig.maxPromptChars`** (engine-level; no CLI flag yet) — hard cap on verbatim prior-turn content. When the cap would be exceeded, oldest turns are dropped first; the most recent turn is always retained. Truncation is **opt-in** — omitting `contextConfig` (or omitting `maxPromptChars`) preserves the legacy passthrough behaviour with no implicit cap.
  - Under any `contextConfig` the orchestrator re-plans the strategy _per turn_ so `same-round` and `recent` policies see turns that completed earlier in the same round. Without `contextConfig` the strategy is still planned once per round (legacy behaviour).
- **Rich Ink Terminal UI** (ROADMAP §3.4) — React + Ink-based renderer that auto-activates when stdout is a TTY. Each expert is assigned a stable color from a 6-color palette (cyan, yellow, magenta, green, blue, red) so the same expert renders in the same color across every round. Components include a `PanelRoster` (assembled experts), `RoundHeader` (round separator), `ExpertCard` (colored name header), `StreamingText` (live turn body with caret cursor while writing), `RetryIndicator` (`ink-spinner` + reason while a turn is being retried), `CostIndicator` (premium-request budget), and a completion banner on `debate.end`. The selector lives in `src/cli/renderers/select.ts` and exposes a new `--format auto` value (the new default). Explicit `--format json` and `--format plain` still always override TTY detection so piped output stays deterministic. Adds `ink`, `ink-spinner`, and `react` as runtime dependencies.
- **`council convene --strategy <name>`** and **`council resume --continue --strategy <name>`** (ROADMAP §2.3, closes #212) — wires `ModeratorStrategy` into the freeform debate orchestrator. Supported values: `round-robin` (default), `devils-advocate`, `consensus-check`. The `devils-advocate` strategy accepts an optional `:<expert-slug>` suffix to pin the contrarian (e.g. `--strategy devils-advocate:senior`); when omitted, the first expert is the advocate. The strategy controls per-round turn order _and_ per-turn prompts: round 0 gets the strategy's framing, round 1+ receives a `Prior discussion:` block built from earlier turn content. `consensus-check` can also terminate a debate early via `shouldContinue()` → `debate.end { reason: "consensus" }`. Strategies are ignored in `--mode structured` (phase-prompts continue to drive that mode). The chosen strategy name is persisted to `debates.moderator`.
- **`DebateConfig.strategy?: ModeratorStrategy`** — pluggable hook on the `Debate` orchestrator's freeform path. Defaults to `createRoundRobinStrategy()` so existing callers see the same structural event sequence.
- **`council conclude [panel] --engine <kind> [--format json|plain]`** (ROADMAP §2.7) — synthesizes the latest debate of a panel into a structured decision framework (consensus, tensions, decision matrix, recommendation, confidence). Reads transcript from the local DB, runs a single moderator-style synthesis prompt through a transient "synthesizer" expert, parses the JSON response (tolerates `json` fences and prose-wrapped JSON), validates against a Zod schema, and renders plain or JSON. Defaults to the most recently-created panel when no name is given. Transcript content is delimited in `<transcript>` tags and capped at 50 turns / 50,000 chars to mitigate prompt injection and unbounded context. Synthesis call uses a 60s `AbortSignal.timeout` so a hung engine fails fast. Warnings are emitted when the latest debate isn't `completed` or when the transcript is truncated.
- **Expert memory recall** (ROADMAP §3.1, second half) — `recallMemory()` reads an expert's prior turns from the local SQLite store and synthesises an `ExpertMemory` (positions, updatedPriors, unresolved) that the prompt-builder renders into Section [7] of the system prompt. `convene` recalls from the most recent prior panel sharing the same template; `resume --continue` patches the persisted system prompt via `applyRecalledMemory()`. Recall is per-expert-slug and parallelised across the roster (`Promise.all`).
- **Prompt-injection hardening for recalled memory** — `sanitizeMemorySnippet()` strips section-header markers (`[N] ...`) and flattens line breaks from every recalled snippet before it reaches the prompt. `applyRecalledMemory()` uses `lastIndexOf("\n[8] CURRENT TASK")` so injected copies of the marker cannot extend the section boundary.
- **Panel auto-composition** (ROADMAP §2.5) — `autoComposePanel(topic, engine, options?)` in `src/core/auto-compose.ts` registers a temporary "composer" expert primed with a meta-prompt, streams its JSON response, strips optional Markdown code fences, parses, and validates against `PanelDefinitionSchema`. Cleans up the composer expert via `finally` even on failure. Configurable `minExperts`/`maxExperts`/`defaultModel`. Surfaces engine error events, empty responses, malformed JSON, and Zod validation failures as descriptive errors. Foundation for `council convene "topic"` without `--template`.
- **`council convene --human "<name>"`** (ROADMAP §3.3) — adds a human participant to the panel. Repeatable (`--human Alice --human Bob`). Human participants take turns alongside AI experts but input is collected interactively instead of via the engine. Human turns are persisted with `speakerKind: "human"` and do NOT count as premium requests in cost tracking. PlainRenderer shows `[You]` prefix for human turns.
- **`speakerKind` on DebateEvents** — `turn.start`, `turn.delta`, `turn.end` now carry optional `speakerKind: "expert" | "human"` (defaults to `"expert"` for backwards compatibility). `DebatePersister` reads this from events instead of hardcoding.
- **`HumanInputProvider` interface** (`src/core/human-input.ts`) — abstraction for collecting human input during debates. Returns `{ kind: "submitted", content }` or `{ kind: "cancelled", reason? }`. CLI implements via readline; tests inject canned responses.
- **`council ask <panel> "<question>" [--expert <slug>] [--engine <kind>]`** — one-shot single-expert chat. Picks one expert from an existing panel (by slug or default to first), runs a 1-round 1-expert debate through the full pipeline (engine → Debate → DebatePersister → Renderer), and returns. Creates a new debate row per ask visible in `resume`, `export`, `memory`. Uses `formatEngineError` for actionable CLI messages. Closes the last vapor-feature in the README.
- **Retry on recoverable engine errors** (ROADMAP §3.7) — `Debate.#runTurn()` now automatically retries up to 2× with exponential backoff (`250ms`, `1s`) when `engine.send()` yields an `error` event with `recoverable: true` (RATE_LIMITED, NETWORK). Non-recoverable errors fail fast. Retry exhaustion emits a final error event but does NOT terminate the debate — the orchestrator moves on to the next expert.
- **`turn.retry` `DebateEvent` variant** — `{ kind: "turn.retry"; expertSlug; attempt; reason }`. Renderers MAY display `[expert retrying…]`; if ignored, downstream behavior is identical to a slightly slower turn. Persistence (`DebatePersister`) does NOT write a row for retries — only the eventual `turn.end` (success) or `error` (exhaustion).
- **`DebateConfig.retryBackoffMs?: readonly number[]`** — list of backoff delays in ms. Length determines max retries. Default `[250, 1000]`. Tests pass `[1, 2]` for fast suite runs.
- **`src/cli/error-mapper.ts`** — `formatEngineError(err)` shared helper that maps `EngineErrorCode` to actionable user messages: NOT_AUTHENTICATED → "run `gh auth login`", MODEL_UNAVAILABLE → "model X isn't available; check `council doctor`", NETWORK → "check your connection", RATE_LIMITED → "wait and retry", CONTEXT_OVERFLOW → "smaller `--max-words` or fewer experts", INTERNAL → "file an issue at github.com/pedrofuentes/Council". Closes #133.
- **Wired into `convene` and `resume`** — engine init + render path now wrapped in try/catch that routes errors through `formatEngineError` to stderr before re-throwing. Users see actionable hints instead of raw stack traces.
- New `MockEngine.failOnSend?: { expertId, afterN, failures, code, message }` test seam — simulates per-send transient failures for retry testing.
- **`council memory list/inspect/reset`** (ROADMAP §3.5) — three subcommands for inspecting and curating Council's local SQLite state at the panel/expert/debate/turn level.
  - **`memory list`**: per-panel summary (name, expert/debate/turn counts, last activity timestamp). `--panel <name>` filters to one. `--format json|plain`.
  - **`memory inspect <panel>`**: detailed view (topic, latest debate prompt + status + turn count, expert displayNames + models). `--expert <slug>` focuses on one expert and shows their (truncated, 600 chars) system prompt, per-expert turn count, and recalled memory (positions / updated priors / unresolved questions extracted from prior turns).
  - **`memory reset <panel> --yes`**: destructive. **Requires `--yes` flag explicitly** (no interactive prompt — flag-only safety gate). Default mode deletes debates + turns, keeps panel + experts. `--hard` deletes the panel entirely (FK CASCADE removes everything). `--expert <slug>` drops one expert from the panel.
  - Pure DB read + targeted writes: no engine, no LLM. Wired in as the 7th subcommand.
- **`council export <panel> --format markdown|json|adr [--output <path>]`** (ROADMAP §3.6) — snapshots the latest debate of a panel into a shareable artifact.
  - **markdown** (default): readable transcript with H1 header (panel name + topic), status line, panel roster, per-turn sections grouped by round (expert displayName + model + content as block quotes).
  - **json**: NDJSON identical to `council resume --format json` — same `synthesizeEvents()` helper.
  - **adr**: Architecture Decision Record markdown with Status / Context / Options Considered / Discussion / Decision sections populated from the panel's debate (heuristic: opening turn per expert = position; last turn per expert = synthesis).
  - **`--output <path>`**: writes to file instead of stdout (default: stdout). Confirmation message goes to stderr so stdout stays clean.
- **`src/memory/transcript.ts`** — `loadTranscript(db, panelName)` + `synthesizeEvents(doc)` extracted from `resume`. Pure read path: no engine, no LLM, no persistence side effects. Shared by `resume` and `export` so both commands produce identical event streams from the same DB rows.
- `TranscriptDocument` type — `{ panel, experts, latestDebate, turns }`.
- **`council resume <panel>`** (ROADMAP §3.2) — reopens an existing panel.
  - **Transcript mode** (default): replays the most recent debate's persisted turns as a synthesized `DebateEvent` stream and renders via JSON or Plain. No engine, no LLM calls — pure DB read.
  - **Continue mode** (`--continue "<prompt>"`): runs a NEW debate against the same panel/experts using the existing convene wiring (engine + Debate + DebatePersister + Renderer). Honors the panel's persisted `mode` (freeform or structured) from `configJson`. Reuses the panel's stored expert system prompts and patches Section [7] MEMORY with freshly-recalled memory (§3.1 second half).
  - Engine selection mirrors convene: `--engine mock|copilot` is **required** when `--continue` is set (no silent default — production users never get fake debates persisted as real). Loud `[MOCK ENGINE]` warning to stderr fires only when `mock` is explicitly chosen.
  - Other options: `--format json|plain`, `--max-rounds`, `--max-words`.
  - Resolves panel name → most-recently-created match via `PanelRepository.findByName()`.
- `PanelRepository.findByName(name)` — look up the most-recently-created panel by name. Used by `council resume`. Returns `undefined` if no match.

Anti-sycophancy defaults (`DEFAULT_FORBIDDEN_PHRASES`, `DEFAULT_DEBATE_PROTOCOL`, `DEFAULT_OUTPUT_CONTRACT`) are always injected; profiles may supplement but cannot remove them.

With `mode: "structured"`, the orchestrator runs a fixed 4-phase choreography (opening → cross-examination → rebuttal → synthesis) regardless of `maxRounds`. ### Changed

- ADR-005 supersedes the implicit `better-sqlite3` choice from ADR-002 / ROADMAP §1.7. The persistence backend is now `@libsql/client` (pure WASM) + `@libsql/kysely-libsql`. Rationale: `better-sqlite3` requires native build tools and lacks Node 25.5.0 prebuilds, breaking "simple to run". libsql is pure JS, has an official Kysely dialect, and is API-compatible with Turso (Council Cloud Phase 5).
- `council convene --engine copilot` now wires the real `CopilotEngine` (previously threw "not yet wired"). Integration tests gated by `COUNCIL_INTEGRATION=1` env var (off by default for fast unit-only runs).
- `makeEngineFromKind(kind)` exported from `src/cli/commands/convene.ts` so unit tests can verify wiring without invoking the engine.

### Fixed

- **`council memory reset` (default mode) is now atomic (#403)** — the default reset path previously deleted every debate and then cleared each expert's `extractedMemoryJson` as independent statements with no surrounding transaction, so a failure between or during either loop left the panel half-reset (debates gone but expert memory preserved, or vice versa) with no recovery. Both loops are now wrapped in a single `BEGIN`/`COMMIT`/`ROLLBACK` on the libsql connection (matching the convention used by `document-repository.clearForRetrain` and the documents indexer — Kysely's `db.transaction()` reconnects libsql and drops FTS5 virtual tables on `:memory:`). The user-facing confirmation lines are now emitted only after a successful `COMMIT`, so a rolled-back reset is never partially announced. If `ROLLBACK` itself fails, the original error is still rethrown and a `!! ROLLBACK failed …` line is written to stderr so operators know the DB may need a manual check. Implementation: `buildResetCommand` in `src/cli/commands/memory.ts`.
- **Persona profile analyzer: existingProfile fields injected unsanitized (#358, #366)**— `analyzeDocuments()` now runs all `existingProfile` fields (`communicationStyle`, `decisionPatterns`, `biases`, `vocabulary`, `epistemicStance`) through the shared `sanitizePromptField` (extracted to `src/core/prompt-sanitize.ts` and reused by `src/core/prompt-builder.ts`) before interpolating them into the analyzer meta-prompt. C0 control bytes are stripped, Unicode line breaks (NEL / U+2028 / U+2029) and CR/LF runs are collapsed to a single space, bracketed numeric section markers (`[N]`) are defanged to `(sec-N)`, and each field is length-capped at 2000 characters. The fence-specific `<` → `&lt;` escape is layered on top so a persisted field still cannot prematurely close the `<documents>` data fence. Adversarial regression tests cover fence-break document content, `[N]`-marker spoofing, runaway-length fields, and C0/Unicode-line-break payloads.
- **`DebatePersister` single-use enforcement (#120)** — calling `persist()` a second time on the same instance now throws with a clear "single-use" message instead of silently overwriting `#debateId` and leaking `#pendingTurnPosition` state across debates. Construct a fresh persister per debate.
- **`DebatePersister` orphan turn.end warning (#119)** — `turn.end` events with no matching prior `turn.start` (orchestrator protocol violations) now call the optional `deps.logger.warn(...)` if a logger is configured. When no logger is provided, drops silently as before (backwards-compatible default).
- New `DebatePersisterLogger` interface (`{ warn(message: string): void }`) — minimal injectable surface for warnings; production callers can pass a console-backed implementation, tests pass a Vitest mock.
- **`DebatePersister.persist()`** no longer silently mutates the debate row to `status='aborted'` when the terminal `debate.end` update itself throws. Renamed `normalEnd` to `terminalUpdateAttempted` and set it BEFORE the await — so a failed terminal update lets the original error bubble (truthful) and leaves the row at its pre-attempt state (`running`). Resolves #150.
- **`convene` partial-failure rollback** now filters to only the experts whose `addExpert` actually fulfilled, instead of sweeping the whole experts list and relying on unverified `removeExpert`-idempotency. Adds an explicit JSDoc contract on `CouncilEngine.removeExpert` codifying the no-op-for-unknown-ids guarantee + a regression test against `MockEngine`. Resolves #151.
- `MockEngine.failOnAddExpert.afterN` semantics changed: now fails ALL calls after the Nth (was: only the (N+1)th). Sharper test signal — `afterN: 1` with 4 experts now means 1 fulfilled + 3 rejected, not 1+1+2.
- `MockEngine.removeExpertCalls: readonly string[]` — new test-only accessor capturing every `removeExpert(id)` invocation in temporal order.
- `convene` MOCK warning now writes to a separate `writeError` channel (stderr), keeping `--format json` output as pure NDJSON on stdout. Resolves #127.
- `makeEngineFromKind` now has an exhaustive `default` arm that throws on unknown engine kind (previously silently returned `undefined`). Resolves #128 and #134.
- `convene` `--engine` validation now uses validate-then-assign (no silent ternary fallback). Resolves #129.
- `convene` cleanup errors (engine.stop / db.destroy) are now surfaced via `writeError` with a `!! ... failed during cleanup:` prefix instead of being silently swallowed. Resolves #130.
- `convene` registers experts in parallel via `Promise.all` instead of sequentially, cutting startup latency from O(N × session-create-ms) to O(1). Resolves #131.
- `bin/council.ts` JSDoc updated — engine is no longer "mock by default". Resolves #126.
- `makeEngineFromKind` JSDoc tagged `@internal` per stability convention. Resolves #135.
- Removed redundant `if (!INTEGRATION) describe(...)` fallback in `tests/integration/convene-copilot.test.ts`. Resolves #136.
- **CopilotEngine + MockEngine** now expose a `lastStopErrors: readonly Error[]` getter populated during `stop()`. Per-session disconnect failures (and the final `client.stop()` failure) are aggregated here instead of being silently swallowed. Backwards-compatible: `stop()` still returns `Promise<void>`. Resolves #143.
- **`convene` parallel addExpert** is now leak-safe via `Promise.allSettled` + rollback. If any `addExpert` rejects, every successfully-created expert is removed via `engine.removeExpert()` before re-throwing, so no `CopilotSession` survives past cleanup. Error message: `could not register all experts (X/Y failed): <first reason>`. Resolves #142.
- **`DebatePersister.persist()`** now self-finalizes on abnormal exit. If the source stream throws or the consumer breaks the for-await loop, the debate row is marked `status='aborted'` with `endedAt` set in a `finally` block. This unblocks ROADMAP §3.2 session-resume which needs to distinguish abandoned from running debates. Resolves #117.
