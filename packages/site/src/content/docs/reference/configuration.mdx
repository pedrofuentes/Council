Council configuration lives at `~/.council/config.yaml`. All settings are optional — Council ships with sensible defaults designed for first-run safety.

## Config File Location

```bash
~/.council/config.yaml
```

Create or edit:

```bash
council config set <key> <value>
```

View current config:

```bash
council config get [key]
```

## Schema reference

:::tip[Canonical reference]
The complete, authoritative list of every configuration key — its dot-path, type, default, and description — is **auto-generated from the CLI's Zod config schema** so it can never drift from the code. See the **[Config & Environment Reference](/Council/reference/config-reference/)**.
:::

All keys are optional and grouped into sections — `defaults`, `telemetry`, `providers`, `expert`, `documents`, `chat`, `conclude`, and `paths`. A typical `config.yaml` overriding a few common settings looks like:

```yaml
defaults:
  maxRounds: 5
  maxExperts: 4
documents:
  aiExtraction: ask
telemetry:
  enabled: false
```

## Editing Config

### Set a value

```bash
council config set defaults.maxRounds 5
council config set documents.aiExtraction ask
```

### Get a value

```bash
council config get defaults.maxRounds
```

### View all settings

```bash
council config get
```
