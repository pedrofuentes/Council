# Support

## Getting Help

Council is a maintainer-driven open-source project. Support is provided on a
best-effort basis.

### Reporting Issues

- **Bug reports**: Open a [GitHub Issue](https://github.com/pedrofuentes/Council/issues/new)
  using the bug template. Include steps to reproduce, expected vs. actual
  behavior, and your environment details (`council --version`, Node.js version,
  OS).
- **Feature requests**: Open a [GitHub Issue](https://github.com/pedrofuentes/Council/issues/new)
  describing the use case and proposed solution.

### Security Vulnerabilities

**Do not report security issues publicly.** Use GitHub's private vulnerability
reporting:

1. Navigate to the [Security tab](https://github.com/pedrofuentes/Council/security).
2. Click **Report a vulnerability** to open a private Security Advisory.
3. Follow the guidance in [SECURITY.md](./SECURITY.md).

### Response Expectations

- We aim to acknowledge new issues and security reports within a few business
  days.
- Responses and fixes are provided on a best-effort basis. Pre-1.0 software —
  breaking changes may occur in minor releases (see [RELEASING.md](./RELEASING.md)).
- For general questions, check existing issues or open a new one. GitHub
  Discussions may be enabled in the future.

### Contributing

Contributions are welcome! See [CONTRIBUTING.md](./CONTRIBUTING.md) for
guidelines on submitting pull requests.

Thank you for using Council.
